============
贡献指南
============

首先，**感谢您**来到这里并考虑为 Odoo 做贡献！

本指南为初次贡献者和资深贡献者提供指引。作为一个开放核心项目，Odoo 接受来自所有人的贡献，包括 Odoo 员工和 Odoo 社区的成员。Odoo 社区由用户、开发人员和其他贡献时间与专业知识的相关方组成。

根据您的技能和兴趣，有多种方式可以为 Odoo 做出贡献。下表列出了其中最重要的一些方式。

.. list-table::
   :header-rows: 1
   :stub-columns: 1
   :widths: 20 35 25 20

   * - 贡献方式
     - 描述
     - 资源
     - 目标人群
   * - 加入社区
     - 参与论坛讨论，帮助回答问题，并协助 Odoo 社区的其他成员。
     - * `Odoo 论坛 <https://www.odoo.com/forum>`_
     - 对 Odoo 感兴趣的任何人
   * - 提交错误报告
     - 通过报告您在 :doc:`支持的版本 </administration/supported_versions>` 中发现的错误，帮助提升 Odoo 的稳定性。确保尽可能包含详细信息（版本、环境、测试场景等）。如果您没有 Odoo 企业版所需的支持许可，请在 Odoo 的 GitHub 仓库上创建一个新问题。
     - * `Odoo 支持 <https://www.odoo.com/help>`_
       * `GitHub 问题追踪器 <https://github.com/odoo/odoo/issues>`_
     - Odoo 用户
   * - 翻译 Odoo
     - 将 Odoo 的用户界面和文档翻译成不同语言。
     - * `Odoo 在 Transifex 上的项目 <https://explore.transifex.com/odoo/>`_
     - 精通多种语言的任何人
   * - 贡献代码库
     - 向 Odoo 的 GitHub 仓库提交 pull request，修复错误、添加新功能或提高产品的性能和可靠性。您还可以审阅和评论其他贡献者的 pull request。通过为代码库做出贡献，您可以对 Odoo 的质量和功能产生直接而持久的影响。
     - * :doc:`指南：贡献代码库 <contributing/development>`
       * :doc:`开发者教程 </developer/tutorials>`
       * :doc:`操作指南 </developer/howtos>`
     - 有 Python 和 Web 开发经验的开发者
   * - 贡献文档
     - 提交对 Odoo 文档的修改和改进。由于产品更新迅速，及时在文档中反映这些变化可能具有挑战性。通过更新文档，您可以帮助许多用户更好地使用 Odoo。
     - * :doc:`指南：贡献文档 <contributing/documentation>`
     - 对改进 Odoo 文档感兴趣的任何人
   * - 创建自定义模块
     - 开发新模块以扩展 Odoo 的功能。自定义模块可以在 Odoo 应用商店或其他 Odoo 模块库中免费或付费提供。通过创建自定义模块并将其提供给他人，您可以帮助壮大 Odoo 生态系统，并为平台用户提供更多的价值。
     - * `Odoo 应用商店 <https://apps.odoo.com/apps>`_
     - 有 Python 和 Web 开发经验的开发者

.. important::
   Odoo 员工主要负责开发代码库的新功能并撰写文档的新内容。请确保在开始工作之前与 Odoo 团队讨论任何新功能或页面的想法，以便他们为您提供绿灯。他们可能已经在开发某些功能，只有批准的功能才能被合并。
